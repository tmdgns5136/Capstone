# 출석 관리 시스템 - 전체 기능 테스트 가이드라인

## 테스트 계정 정보

| 역할 | 아이디 | 비밀번호 |
|------|--------|----------|
| 관리자 | admin | test1234 |
| 교수 | 100001 (김영수) | test1234 |
| 교수 | 100002 (이정희) | test1234 |
| 교수 | 100003 (박민수) | test1234 |
| 학생 | 202100001 (홍길동) | test1234 |
| 학생 | 202100006 (정하늘) | test1234 |

> 학생 202100001~202100005는 프로그래밍기초/데이터베이스/컴퓨터네트워크 수강
> 학생 202100006~202100010은 운영체제/소프트웨어공학/임베디드시스템 수강

## 1. 공통 기능

### 1-1. 로그인/로그아웃
- [ ] 학생 계정으로 로그인 → 학생 대시보드 이동 확인
- [ ] 교수 계정으로 로그인 → 교수 대시보드 이동 확인
- [ ] 관리자 계정으로 로그인 → 관리자 대시보드 이동 확인
- [ ] 잘못된 비밀번호 입력 → 에러 메시지 확인
- [ ] 존재하지 않는 아이디 입력 → 에러 메시지 확인
- [ ] 로그아웃 버튼 클릭 → 로그인 페이지로 이동 확인
- [ ] 아이디 저장 체크 후 로그인 → 재방문 시 아이디 유지 확인

### 1-2. 회원가입
- [ ] 학생 회원가입: 학번(9자리), 이름, 이메일, 전화번호, 비밀번호, 얼굴 사진 3장(좌/정면/우) 입력
- [ ] 교수 회원가입: 사번(6자리), 이름, 이메일, 전공, 전화번호, 비밀번호 입력
- [ ] 이메일 인증코드 발송 및 확인
- [ ] 중복 학번/사번으로 가입 시도 → 에러 확인
- [ ] 필수 입력값 누락 시 → 유효성 검증 확인

### 1-3. 비밀번호 찾기
- [ ] 이메일 입력 → 인증코드 발송
- [ ] 인증코드 입력 → 새 비밀번호 설정
- [ ] 변경된 비밀번호로 로그인 확인

## 2. 학생 기능 (202100001 로그인)

### 2-1. 홈/대시보드
- [ ] 오늘의 강의 목록 표시 확인 (요일에 맞는 강의가 나오는지)
- [ ] 현재 진행 중인 강의 상태 표시 확인
- [ ] 출석 현황 요약 확인

### 2-2. 시간표
- [ ] 주간 시간표 그리드(9AM~6PM) 표시 확인
- [ ] 수강 중인 3과목이 올바른 요일/시간에 배치되는지 확인
  - 프로그래밍기초: 월 09:00~12:00 (3시간)
  - 데이터베이스: 화 09:00~12:00 (3시간)
  - 컴퓨터네트워크: 수 14:00~17:00 (3시간)

### 2-3. 수강 과목 목록
- [ ] 학기 필터(2026년 1학기) 선택 → 수강 과목 3개 표시 확인
- [ ] 각 과목 클릭 → 과목 상세 페이지 이동 확인

### 2-4. 과목 상세 (프로그래밍기초 클릭)
- [ ] 강의 정보 표시 확인 (교수명, 강의실, 시간)
- [ ] 회차별 출석 기록 목록 확인 (3회차 출석 데이터)
- [ ] 공지사항 탭 → 공지 목록 확인 (2건)
- [ ] 공지 클릭 → 상세 내용 확인
- [ ] Q&A 탭 → 질문 목록 확인
- [ ] 질문 상세 → 답변 표시 확인
- [ ] 새 질문 작성 (제목, 내용, 비공개 여부)
- [ ] 비공개 질문은 본인만 볼 수 있는지 확인

### 2-5. 출석 통계
- [ ] 전체 출석률 표시 확인
- [ ] 과목별 출석/지각/결석 횟수 확인
- [ ] 차트/그래프 표시 확인

### 2-6. 공결 신청
- [ ] 공결 신청서 작성 (제목, 사유, 증빙 파일 첨부)
- [ ] 신청 후 목록에서 상태(PENDING) 확인
- [ ] 기존 신청 내역 조회 확인
- [ ] 대기 중인 신청 수정/삭제 가능 확인

### 2-7. 이의 신청
- [ ] 이의 신청서 작성 (제목, 사유, 증빙 파일 첨부)
- [ ] 신청 후 목록에서 상태 확인
- [ ] 반려된 신청의 반려 사유 확인

### 2-8. 프로필
- [ ] 내 정보 표시 확인 (학번, 이름, 이메일, 전화번호)
- [ ] 전화번호 변경 → 저장 확인
- [ ] 비밀번호 변경 (현재 비밀번호 확인 → 새 비밀번호 설정)
- [ ] 프로필 사진 변경 요청 (좌/정면/우 3장 업로드)
- [ ] 사진 변경 요청 내역 조회
- [ ] 회원 탈퇴

### 2-9. 알림
- [ ] 알림 목록 표시 확인 (공지, 답변, 공결/이의 결과)
- [ ] 알림 클릭 → 읽음 처리 확인
- [ ] 알림 벨 아이콘에 읽지 않은 알림 개수 표시

## 3. 교수 기능 (100001 로그인)

### 3-1. 홈/대시보드
- [ ] 담당 강의 수, 전체 학생 수, 오늘 출석률 등 통계 확인
- [ ] 오늘의 강의 목록 표시 확인
- [ ] 대기 중인 공결/이의 신청 건수 확인

### 3-2. 강의 목록
- [ ] 학기 필터 → 담당 강의 2개 표시 확인
  - 프로그래밍기초 (월 09:00~12:00)
  - 운영체제 (목 13:00~16:00)
- [ ] 강의 클릭 → 상세 페이지 이동

### 3-3. 강의 상세 - 출석 탭
- [ ] 회차별 출석 현황 표시 확인
- [ ] 날짜 선택 → 해당 회차 학생별 출석 상태 확인
- [ ] 출석 상태 수동 변경 (출석↔지각↔결석) → 저장

### 3-4. 강의 상세 - 공지 탭
- [ ] 공지사항 목록 표시 확인
- [ ] 새 공지 작성 (제목, 내용)
- [ ] 작성한 공지 목록에 표시 확인

### 3-5. 강의 상세 - Q&A 탭
- [ ] 질문 목록 표시 확인
- [ ] 질문 클릭 → 상세 내용 확인
- [ ] 답변 작성 → 저장 확인
- [ ] 기존 답변 수정/삭제
- [ ] 비공개 질문도 교수는 볼 수 있는지 확인

### 3-6. 수업 시작/종료
- [ ] 오늘 수업 목록에서 "수업 시작" 클릭
- [ ] 수업 상태가 IN_PROGRESS로 변경 확인
- [ ] "수업 종료" 클릭 → ENDED로 변경 확인
- [ ] 수업 시작 시 해당 세션에 학생별 출석 레코드(TBD) 생성 확인

### 3-7. 실시간 모니터링
- [ ] 진행 중인 수업의 학생 출석 현황 실시간 확인
- [ ] 입실/퇴실 시간, 체류율 표시 확인

### 3-8. 공결 신청 관리
- [ ] 대기 중인 공결 신청 목록 확인
- [ ] 신청 상세 확인 (학생 정보, 사유, 증빙 파일)
- [ ] 증빙 파일 다운로드
- [ ] 승인 처리 → 상태 변경 확인
- [ ] 반려 처리 (반려 사유 입력) → 상태 변경 확인

### 3-9. 이의 신청 관리
- [ ] 대기 중인 이의 신청 목록 확인
- [ ] 신청 상세 확인
- [ ] 승인 처리 → 출석 상태 변경 확인
- [ ] 반려 처리 (반려 사유 입력)

### 3-10. 프로필
- [ ] 내 정보 표시 확인 (사번, 이름, 이메일, 전공, 전화번호)
- [ ] 전화번호 변경
- [ ] 비밀번호 변경

### 3-11. 알림
- [ ] 공결/이의 신청 알림 표시 확인
- [ ] 알림 읽음 처리

## 4. 관리자 기능 (admin001 로그인)

### 4-1. 홈/대시보드
- [ ] 시스템 전체 통계 표시 확인 (총 학생수, 교수수, 강의수 등)

### 4-2. 강의 관리
- [ ] 교수 사번으로 검색 → 해당 교수의 강의 목록 확인
- [ ] 새 강의 등록 (강의코드, 강의명, 강의실, 연도, 학기, 분반, 시간, 요일, 교수)
- [ ] 기존 강의 수정
- [ ] 강의 삭제
- [ ] 강의에 학생 추가 (학번으로)
- [ ] 강의의 수강생 목록 조회
- [ ] 수강생 삭제

### 4-3. 학생 관리
- [ ] 전체 학생 목록 조회 (페이지네이션)
- [ ] 학생 등록 (학번, 이름, 이메일, 전화번호, 비밀번호)
- [ ] 학번으로 학생 검색 → 상세 정보 확인
- [ ] 학생 정보 수정
- [ ] 학생 삭제

### 4-4. 교수 관리
- [ ] 전체 교수 목록 조회 (페이지네이션)
- [ ] 교수 등록 (사번, 이름, 이메일, 전공, 전화번호, 비밀번호)
- [ ] 사번으로 교수 검색 → 상세 정보 확인
- [ ] 교수 정보 수정
- [ ] 교수 삭제

### 4-5. 프로필 사진 승인 관리
- [ ] 대기 중인 사진 변경 요청 목록 확인
- [ ] 요청 상세 확인 (학생 정보, 좌/정면/우 사진)
- [ ] 승인 처리
- [ ] 반려 처리 (반려 사유 입력)
- [ ] 처리 완료된 요청 목록 확인

### 4-6. 알림
- [ ] 관리자 알림 확인

## 5. 크로스 기능 테스트 (기능 간 연동)

### 5-1. 공결 신청 → 승인 흐름
1. [ ] 학생(202100001)으로 로그인 → 프로그래밍기초 공결 신청
2. [ ] 교수(100001)로 로그인 → 공결 신청 알림 확인
3. [ ] 공결 신청 승인 처리
4. [ ] 학생으로 다시 로그인 → 승인 알림 확인
5. [ ] 해당 회차 출석 상태 변경 확인

### 5-2. 이의 신청 → 반려 흐름
1. [ ] 학생으로 로그인 → 이의 신청 작성
2. [ ] 교수로 로그인 → 이의 신청 반려 (사유 입력)
3. [ ] 학생으로 로그인 → 반려 알림 및 사유 확인

### 5-3. 수업 진행 흐름
1. [ ] 교수(100001)로 로그인 → 수업 시작
2. [ ] 학생(202100001)으로 로그인 → 현재 수업 상태 확인
3. [ ] 교수로 로그인 → 모니터링에서 학생 출석 현황 확인
4. [ ] 교수로 출석 수동 변경
5. [ ] 교수로 수업 종료
6. [ ] 학생으로 로그인 → 출석 기록 반영 확인

### 5-4. 관리자 → 강의/학생 관리 흐름
1. [ ] 관리자로 로그인 → 새 강의 등록
2. [ ] 강의에 학생 추가
3. [ ] 해당 학생으로 로그인 → 수강 과목에 새 강의 표시 확인
4. [ ] 관리자로 학생 삭제 → 해당 계정 로그인 불가 확인

### 5-5. Q&A 흐름
1. [ ] 학생으로 로그인 → 질문 작성
2. [ ] 교수로 로그인 → 질문 확인 및 답변 작성
3. [ ] 학생으로 로그인 → 답변 알림 확인 및 답변 내용 확인

## 6. 예외/에러 테스트

- [ ] 로그인 없이 학생/교수/관리자 페이지 URL 직접 접근 → 로그인 페이지 리다이렉트
- [ ] 학생 계정으로 교수/관리자 페이지 접근 시도 → 차단 확인
- [ ] 교수 계정으로 관리자 페이지 접근 시도 → 차단 확인
- [ ] 토큰 만료 후 API 호출 → 자동 토큰 갱신 또는 재로그인
- [ ] 존재하지 않는 강의 ID로 URL 접근 → 에러 처리 확인
- [ ] 파일 업로드 시 10MB 초과 파일 → 에러 메시지 확인

## 테스트 팁

1. **브라우저 개발자 도구(F12)** 의 Network 탭을 열어두면 API 요청/응답을 실시간으로 확인 가능
2. **Console 탭**에서 프론트엔드 에러 확인
3. 백엔드 터미널에서 Spring Boot 로그 확인
4. 문제 발생 시 **요청 URL, HTTP 상태코드, 응답 body**를 기록
5. 오늘(목요일) 기준 수업 있는 강의:
   - 운영체제 (목 13:00~16:00)
6. 전체 강의 시간표:
   - 월: 프로그래밍기초 09:00~12:00
   - 화: 데이터베이스 09:00~12:00
   - 수: 소프트웨어공학 10:00~12:00, 컴퓨터네트워크 14:00~17:00
   - 목: 운영체제 13:00~16:00
   - 금: 임베디드시스템 13:00~15:00
